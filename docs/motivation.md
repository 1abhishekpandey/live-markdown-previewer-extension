# Why LiveMarkdown?

VS Code handles markdown well — but the default experience has friction that adds up over time.

# The problem with VS Code's built-in preview

**Split-screen preview** — The default preview (`Cmd+Shift+V`) opens in a side panel, splitting your workspace in half. You lose half your editing space and constantly shift focus between the source and preview panes.

**Manual toggle** — VS Code offers an inline preview mode, but you have to trigger it every time you open a file.

**Read-only preview** — Even when the preview is open, it is read-only. You cannot click into it and start editing like you would in Notion. Editing still happens in the raw source pane.

# What LiveMarkdown does differently

LiveMarkdown replaces the default text view entirely. When you open a `.md` file, you see rendered content — headings, links, tables, code blocks, task lists — all formatted inline. Click anywhere and start typing. Changes are reflected live with no extra steps.

- **No split pane** — the editor and preview are the same surface.
- **No shortcut to remember** — rendered view is the default. Raw markdown is one toggle away (`Shift+Cmd+M`) when you need it.
- **Direct editing** — click, type, and the underlying markdown updates automatically.
- **Copy as raw or rich** — toggle between copying selected content as markdown syntax or rendered HTML, useful when moving content between tools.

# Why now?

Markdown has become the standard format for LLM prompts, project documentation, and knowledge bases. As these workflows grow, constantly switching between raw source and preview mode becomes a daily friction point. LiveMarkdown closes that gap inside the editor you already use.
